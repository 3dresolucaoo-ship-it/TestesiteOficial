`qual = true` — qualquer usuario anonimo faz `GET /rest/v1/inventory` e recebe
TODO o estoque de TODOS os tenants. Sem autenticacao. Sem filtro de project_id.

**Fix:**
```sql
DROP POLICY IF EXISTS inventory_public_read ON public.inventory;
```

### GAP-02 — `products` policy `products_public_read` sem filtro [SEVERIDADE 4]

`qual = true` — qualquer anon le TODOS os produtos de TODOS os tenants.
Dados de custo, margem, preco de fabricacao expostos publicamente.

**Fix:**
```sql
DROP POLICY IF EXISTS products_public_read ON public.products;

-- Permitir leitura apenas de produtos cujo dono tem catalogo publico:
CREATE POLICY products_public_read ON public.products FOR SELECT
  USING (
    EXISTS (
      SELECT 1 FROM public.catalogs c
      WHERE c.user_id = products.user_id
        AND c.is_public = true
    )
  );
```

### GAP-03 — `set_active_payment_config` aceita p_user_id como parametro [SEVERIDADE 3]

SECURITY DEFINER function recebe `p_user_id uuid` do caller em vez de usar `auth.uid()` internamente.
Um usuario autenticado que enumere um `config_id` alheio pode chamar a funcao com outro `p_user_id`
e manipular configuracoes de pagamento de terceiros. IDOR em dados financeiros.

**Fix — recriar sem p_user_id:**
```sql
CREATE OR REPLACE FUNCTION public.set_active_payment_config(p_config_id uuid)
RETURNS void
LANGUAGE plpgsql
SECURITY DEFINER
SET search_path TO 'public'
AS $$
DECLARE
  v_owner UUID;
BEGIN
  SELECT user_id INTO v_owner FROM payment_configs WHERE id = p_config_id;

  IF v_owner IS NULL THEN
    RAISE EXCEPTION 'Payment config % not found', p_config_id;
  END IF;

  IF v_owner <> auth.uid() THEN
    RAISE EXCEPTION 'Unauthorized: config does not belong to caller';
  END IF;

  UPDATE payment_configs SET is_active = false
    WHERE user_id = auth.uid() AND is_active = true AND id <> p_config_id;

  UPDATE payment_configs SET is_active = true WHERE id = p_config_id;
END;
$$;
```
> Lembrar de atualizar chamador em `services/paymentConfig.ts` para remover o argumento user_id.
