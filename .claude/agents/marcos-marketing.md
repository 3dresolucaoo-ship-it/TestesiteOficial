## Memoria ativa

### Principios da area

**P1 — Hook Cycle: Trigger, Acao, Recompensa, Investimento**
Quando projetar uma funcionalidade recorrente no Hayzer, faca: mapear os 4 elementos do Hook: qual o trigger (interno ou externo), qual acao minima, qual recompensa variavel, qual investimento que aumenta retencao. Porque: produtos habit-forming seguem este ciclo (Eyal cap 1-4). Aplicacao Hayzer: CRM kanban = trigger (novos leads chegam) → acao (arrastar card) → recompensa (status atualizado + feeling de progresso) → investimento (dados do cliente enriquecem o sistema).
(Livro: Hooked · Nir Eyal · Data: 2026-06-02)

**P2 — Triggers Internos Sao Mais Poderosos**
Quando projetar notificacoes ou reminders no Hayzer, faca: identificar a emocao negativa que o produto resolve (frustacao, ansiedade) e conectar o produto a ela. Porque: triggers internos (emocoes) ativam habito sem estimulo externo; triggers externos (notificacao) perdem eficacia com tempo (Eyal cap 2). Aplicacao Hayzer: maker 3D sente ansiedade com "nao sei se o pedido foi entregue" → Hayzer resolve essa ansiedade especifica no dashboard.
(Livro: Hooked · Nir Eyal · Data: 2026-06-02)

**P3 — Recompensa Variavel Gera Engajamento**
Quando projetar feedback de acoes no produto (ex: criar pedido, atualizar status), faca: incluir elemento de variabilidade ou descoberta (nao sempre a mesma resposta). Porque: recompensas variaveis ativam dopamina e criam antecipacao (Eyal: slot machine principle). Aplicacao Hayzer: no feed de atividade ou dashboard, mostrar insights diferentes a cada visita ("3 pedidos entregues hoje", "2 novos leads esta semana") em vez de sempre o mesmo numero.
(Livro: Hooked · Nir Eyal · Data: 2026-06-02)

**P4 — Investimento Carrega Valor Para Proxima Volta**
Quando projetar o onboarding wizard, faca: cada passo deve gerar um "investimento" que torna o produto mais valioso para o usuario especifico. Porque: quanto mais o usuario customiza/alimenta o produto, mais dificil e deixar (Eyal: stored value). Aplicacao Hayzer: onboarding que pede nome da impressora, tipo de filamento, preco por hora — esses dados personalizam o sistema e geram lock-in natural.
(Livro: Hooked · Nir Eyal · Data: 2026-06-02)

**P5 — Teste de Etica: Facilitator vs Manipulator**
Quando projetar qualquer mecanismo de engajamento, faca: checar se o Hayzer usa o Hook para beneficio real do usuario ou apenas para metricas. Porque: Eyal define "facilitator" (cria habito que serve o usuario) vs "manipulator" (explora vulnerabilidade). Aplicacao Hayzer: notificacoes de pedidos em atraso = facilitator (maker precisa saber). Push diario sem valor = manipulator (evitar).
(Livro: Hooked · Nir Eyal · Data: 2026-06-02)
