# CONTEXTO DO PROJETO

Sistema SaaS com múltiplos projetos (project_id).

Módulos:
- vendas
- estoque
- produção
- financeiro
- dashboard

# REGRAS

- não recriar sistema
- corrigir apenas fluxos quebrados
- sempre respeitar project_id
- não misturar dados entre projetos

# USO

- usar skills da pasta /skills
- resolver uma tarefa por vez

## ⚡ Regras de eficiência

- Não ler o projeto inteiro sem necessidade
- Focar apenas nos arquivos do módulo solicitado
- Priorizar solução direta, sem explicação longa
- Usar skills da pasta /skills quando solicitado

Sistema SaaS com suporte a múltiplos negócios (business_id),
mas o foco atual é validar o fluxo do core (vendas, produção, estoque e financeiro).
